# Security Policy

Robotics Engineer Environment is an experimental development environment. It
is not a security product, and users are responsible for reviewing commands and
configuration before using them on a development or production machine.

## Sensitive information

Never publish credentials, tokens, SSH keys or hosts, private IP addresses,
company infrastructure, private workspace paths, screenshots containing
sensitive data, or logs containing secrets.

If sensitive data appears in a pull request, issue, screenshot, or commit,
remove it immediately. The owner of any exposed credential must rotate or
revoke it as soon as possible. Removing the text from GitHub does not make an
exposed credential safe to continue using.

## Reporting a security concern

Do not disclose secrets or exploitable private details in a public report. The
project does not currently have a dedicated security email address. Open a
minimal GitHub issue without including the sensitive value, or contact the
maintainer through their GitHub profile:

<https://github.com/jlgalavi>

Include only enough non-sensitive context to establish contact and describe
the affected area. Further handling can then be coordinated safely.
